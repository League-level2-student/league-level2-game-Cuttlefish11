Tower Defense

by: Robert Garibaldi

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I have enjoyed playing tower defense games like Kingdom rush in the past and wanted to create a game emulating it of my own. Additionally, a simple tower defense game was perfect for filling the criteria of this project.

2. What I Learned

I learned some ways that my game was too easy and too hard. While it is only slightly easier at the start of the game(which was too hard before), the end game (which was too easy) is now more fun and challenging because of my peer's and teacher's suggestions. One specific good edit I implemented gave trolls two health instead of one.

3. Architecture of My Game

The object of the game is to get 300 money. To get the money you kill monsters before they get to the bottom of the screen. Archer and mage towers kill monsters, but they cost money which takes time to get and is needed to win. Another part that appears is trolls which come a little later in the game, in addition to the foes at the start. This creates a myriad of classes such as Foe, Troll, Turret, Game Panel, Object Manager, and more. Object Manager and Game Panel are very similar to the classes with those names in League Invaders. This was useful for making the game work, creating objects, and checking collisions between classes.

4. Technical Implementation

One part of the game that I am particularly proud of is the tower placement. To place a tower you must click on a part of the map and then type in a or m for archer or mage tower. For this to work, the GamePanel class and the ObjectManager class have to work together. Moreover, multiple for loops make sure the tower to be placed has to be both in a specific location, and be payable with the player's money. Another aspect I am proud of is the trollSpawn mechanic since it was quite difficult to have the trolls appear when I wanted them to. To achieve this goal I made each foe that spawns reduce a time variable, and once it reaches a certain number the trolls start spawning.

5. Most Challenging Part

The trolls were the most challenging part of the project because in some games they spawned and in others, they didn't. Eventually, I figured out that the trollSpawn timer needs to be completely reset and remade when the game ends. However, it was yet more challenging because that was not the end of my trollSpawn troubles. It turns out I did not add the changes I discovered to the end condition where the player simply clicks enter to change the game state. It took me a while to find and then fix this problem.

6. Future Fixes and Updates

If I was to continue working on this game I would make it take longer, add more monster types, and add more tower options. The additional tower options would be great since they make the game less monotonous and are just so enjoyable to come up with. My brother suggested a treasury tower, and there are definitely more ways to adjust tower attacks. For example, rate of fire, direction, projectile size, etc.).
